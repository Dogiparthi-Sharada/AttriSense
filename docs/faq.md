# FAQ

## General

### Is this real HR data?

No. **100% synthetic**, generated by [`generate_demo_workforce_data.py`](https://github.com/Dogiparthi-Sharada/AttriSense/blob/main/generate_demo_workforce_data.py) with `RANDOM_SEED=42`. Nothing in this repo represents any real person.

### Can I use this in production?

Not as-is. See [Intended Use](ethics/intended-use.md). The model card disclaims real employment-decision use. A real deployment needs a third-party bias audit, an appeal process, and a long checklist of governance work that AttriSense does not provide.

### Why two dashboards?

The original `streamlit_app.py` is the historic interview deliverable, preserved verbatim. `production/streamlit_app.py` is the production-shaped follow-up with 5 additional tabs, a dark theme, 31 unit tests, and Docker support. See [Why two apps?](design/decisions.md#why-two-apps).

### Why three languages in the RAG?

To demonstrate **cross-lingual semantic retrieval** — a Spanish query can return relevant English notes via the multilingual embedding model. Real exit-interview corpora at multinational companies include multiple languages. See [Multilingual RAG](features/multilingual-rag.md).

## Setup

### Do I need an OpenAI key?

**No.** The dashboard runs without one. Tabs that need an LLM gracefully fall back:

- **AI Assistant** → TF-IDF ranker over 50 gold questions.
- **Multilingual RAG** → local hashing embeddings.
- **NL→SQL Eval** → tab is hidden.

The other 7 tabs (Executive, SHAP, Compare, Causal Uplift, Fairness, Alert Mock, Ethics) work fully offline.

### Why Python 3.11+?

`langchain 1.0` and EconML's transitive deps require it. Python 3.10 will fail at install time.

### Can I run this on Windows?

Yes. See [Installation Path A](operations/installation.md#path-a--local-python-recommended-for-development) — the venv activation command differs but everything else works.

### Can I run this offline?

Yes after the first install. The fallback paths for AI features mean the dashboard runs with no network at all.

## Model

### Why Random Forest, not XGBoost?

TreeExplainer SHAP is exact for both. RF requires no GPU and trains in 12 seconds. The AUC delta on this synthetic dataset is < 1pp. See [tech stack](architecture/tech-stack.md#scikit-learn).

### Why these four features?

`tenure_months`, `base_salary`, `department_code`, `manager_tenure_months` are observable in any HRIS today. Adding more features (engagement scores, manager 1:1 cadence, calendar load) is the obvious next step but each new feature has to survive a fairness audit. With four, the audit is tractable.

### Can I retrain on my own data?

Yes. Replace `attrisense_synthetic_hr.csv` with a CSV of the same schema:

```
employee_id,department,tenure_months,base_salary,manager_id,manager_tenure_months,voluntary_turnover,exit_notes
```

Then run `python train_retention_risk_model.py`. The dashboard picks up the new DB on next reload.

For a different schema, edit `train_retention_risk_model.py` — it's a single 486-line script.

### How accurate is it?

Test AUC ~0.85 on the synthetic dataset. **Don't infer real-world accuracy from this number** — synthetic data is by design easier to model than real data. Retrain on your own data to get a real benchmark.

## Fairness

### The fairness audit fails on the demo. Is the model broken?

By design. Manufacturing is seeded with a stronger turnover signal so the audit has a real disparate-impact failure to detect. This demonstrates that the audit *works*, not that the model is unsafe.

For a real deployment, the failure means: **don't use this model on real people until you've reweighted training data, applied a fairness constraint, or audited the upstream signal**.

### Why don't you auto-fix fairness?

Auto-mitigation is a human decision, not a build-time toggle. See [fairness policy](ethics/fairness-policy.md#why-we-dont-auto-mitigate).

### Can the audit run on protected attributes (race, gender)?

Yes — `run_audit(df, dimension="gender")` works as long as the column is in the DataFrame. The synthetic dataset doesn't carry these (synthetic gender data would be misleading), so the demo audits department only. A real deployment imports protected attributes at audit time only and never feeds them to the model.

## Causal uplift

### Are the uplift estimates real?

**No.** They're demoware. Real causal inference needs **observed treatment** in the historical data. The synthetic dataset has none, so the Causal Uplift tab simulates treatment assignment from observable features. The estimates are model outputs, not real causal estimates.

### What would real causal inference look like?

You'd need:

1. Historical retention-action records — who got a salary lift, who got a manager change, when, which employees.
2. Propensity-score matching or doubly-robust estimation (both supported by EconML — same API as T-learner).
3. Validation of assumptions: overlap, no unmeasured confounding, SUTVA.

This is a significant data-engineering project that AttriSense does not include.

## NL→SQL

### What if the LLM generates wrong SQL?

The validator blocks anything that isn't `SELECT/WITH`. The connection enforces `PRAGMA query_only=ON`. So "wrong SQL" can:

- Return zero rows (the LLM picked the wrong filter).
- Raise a SQL syntax error (caught and shown to the user).
- Return correct-looking but semantically wrong rows (this is the nightmare scenario — and exactly what the [Eval Harness](features/nl-sql-eval.md) catches).

### How accurate is the AI Assistant?

The eval harness on `gpt-3.5-turbo` typically reports 40-60% pass rate (exact match + cardinality match across 50 questions). Higher with `gpt-4`, lower without prompt tuning.

The point: **measure it on your own deployment**. Don't trust an AI feature you can't quantify.

## Cost

### How much does this cost to run?

| | Cost |
|---|---|
| Local development | $0 |
| Streamlit Community Cloud | $0 |
| OpenAI usage (100 questions/day) | ~$3/month |
| OpenAI usage (multilingual RAG, 100 queries/day) | ~$0.30/month |

A team running zero AI calls pays zero. The dashboard runs fully without network.

### What's the AWS / GCP cost for a real deployment?

AttriSense itself fits on a single small instance (~$15/month for a `t4g.medium`). The cost in a real deployment is **the data engineering** — extracting from the HRIS, scheduling retraining, integrating with Slack/Teams, building the appeal-process workflow. That's typically the 90% the model sits inside.

## Code

### Where do I start reading?

1. [`production/streamlit_app.py`](https://github.com/Dogiparthi-Sharada/AttriSense/blob/main/production/streamlit_app.py) — the dashboard entry point. 510 lines. Read top-to-bottom.

### Why is the package called `attrisense`?

The product name. It rhymes with "intelligence", which is the whole point.

### Why is the original streamlit_app.py 1,119 lines?

It was a single-file demo. The production version (510 lines) is split across `production/src/attrisense/*.py` — the original is preserved as historical reference.
